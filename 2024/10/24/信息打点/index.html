CDN绕过

CDN有三种加速方式：加速域名、加速区域、加速类型（图片、视频等）

根据这三种方式进行绕过

绕过方式：

1、子域名入手：ping一些子域名，或者不加WWW

2、利用网站漏洞：如果目标站点存在漏洞，这就没办法避免了。例如phpinfo敏感信息泄露、Apache status和Jbossstatus敏感信息泄露、网页源代码泄露、svn信息泄露信、github信息泄露等。
若存在web漏洞，服务器主动与我们发起请求连接，我们也能获取目标站点真实ip。例如xss、ssrf、命令执行反弹shell等

3、历史DNS记录：查询ip与域名历史记录，可能会发现使用CDN之前的目标。

4、邮件服务器

5、全网扫描

找到之后修改host文件，将域名与真实ip进行绑定

本地搭建：

网上教程默认安装，模拟器用雷电或夜神就可以

2、修改文件：C:\Users\18117\.MobSF\config.py
   将ANALYZER_IDENTIFIER = os.getenv('MOBSF_ANALYZER_IDENTIFIER', '192.168.31.223:5555')  这里的ip为本机ip，	5555端口为默认开启的网络ADB监听端口。

使用docker

模拟器设置

打开Windows虚拟机，配置安卓模拟器，以雷电模拟器为例。需要改动：允许system分区可写、开启ROOT权限、ADB改为网络ADB。操作方法：

    允许system分区可写：模拟器设置 -> 性能设置-> system.vmdk可写入
    开启ROOT权限：模拟器设置 -> 其他设置 -> ROOT权限开启
    ADB调试：模拟器设置 -> 其他设置 -> ADB调试改为开启远程连接

kali设置

1、拉取镜像:docker pull opensecurity/mobile-security-framework-mobsf:latest

拉取完毕后 ，运行具有静态和动态分析支持的 MobSF的docker容器，注意：正确修改命令中 MOBSF_ANALYZER_IDENTIFIER=<adb device identifier> 的参数！

<adb device identifier>应该改为您的Windows主机的地址和模拟器的ADB端口，例如：MOBSF_ANALYZER_IDENTIFIER=192.168.31.223:5555。其中192.168.31.223为安装过安卓模拟器的Windows的IP地址，5555端口为默认开启的网络ADB监听端口。

2、docker run -it --rm -p 8000:8000 -p 1337:1337 -e MOBSF_ANALYZER_IDENTIFIER=192.168.31.223:5555 opensecurity/mobile-security-framework-mobsf:latest

小程序整体架构

1.主体结构
    小程序包含一个描述整体程序的 app 和多个描述各自页面的 page。
    一个小程序主体部分(即 app)由三个文件组成，必须放在项目的根目录，如下：
    文件 必需 作用
    app.js 是 小程序逻辑
    app.json 是 小程序公共配置
    app.wxss 否 小程序公共样式表
2.一个小程序页面由四个文件组成，分别是:
    xxx.js 页面逻辑
    xxx.json 页面配置
    xxx.wxml 页面结构
    xxx.wxss 页面样式
3.项目整体目录结构
    pages 页面文件夹
    index 首页
    logs 日志
    utils
    util 工具类(mina 框架自动生成,你也可以建立一个：
    api)
    app.js 入口 js(类似于 java 类中的 main 方法)、全局 js
    app.json 全局配置文件
    app.wxss 全局样式文件
    project.config.json 跟你在详情中勾选的配置一样
    sitemap.json 用来配置小程序及其页面是否允许被微信索引
